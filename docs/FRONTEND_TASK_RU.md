# Задание для Frontend разработчика: Панель управления пользователями

## Обзор задания

Разработайте панель управления пользователями с интеграцией бесконечного скролла, функции поиска и многошаговой формы с условными полями. Продемонстрируйте навыки управления состоянием с помощью Effector, интеграции API с помощью Axios и современной архитектуры React компонентов.

---

## Необходимые технологии

- **React** (JavaScript или TypeScript)
- **Axios** - HTTP клиент для API запросов
- **Effector** - библиотека управления состоянием
- **Mantine** - библиотека React UI компонентов
- **API**: [DummyJSON Users API](https://dummyjson.com/docs/users)

---

## Требования к функциональности

### 1. Страница со списком пользователей

**Бесконечный скролл:**
- Отображайте пользователей в виде сетки или списка
- Начальная загрузка: 10 пользователей
- Загружайте 10 дополнительных пользователей при прокрутке до конца страницы
- Показывайте индикатор загрузки во время получения данных
- Обрабатывайте состояние "больше нет данных" соответствующим образом

**Функциональность поиска:**
- Реализуйте поле ввода поиска
- Поисковый запрос должен быть дебаунцирован (минимум 300ms)
- Поиск по имени пользователя
- Отображайте сообщение "Результаты не найдены" когда поиск пустой
- Функциональность очистки результатов поиска

**Отображение пользователей:**
- Показывайте релевантную информацию пользователя (имя, email, возраст, телефон, аватар если доступен)
- Включите кнопку "Редактировать" на каждой карточке пользователя
- Включите кнопку "Добавить нового пользователя" в верхней части списка

### 2. Многошаговая форма пользователя (режим создания/редактирования)

**Шаг 1: Личная информация**
- Имя (обязательно, минимум 2 символа)
- Фамилия (обязательно, минимум 2 символа)
- Возраст (обязательно, число от 18 до 100)
- Пол (обязательно, выпадающий список: Мужской, Женский, Другое)
- Email (обязательно, валидный формат email)
- Телефон (обязательно)

**Шаг 2: Информация адреса**
- Адрес улицы (обязательно)
- Город (обязательно)
- Штат/Область (опционально)
- Почтовый индекс (обязательно, числовая валидация)
- **Условное поле**:
  - Чекбокс "Международный адрес?"
  - Если отмечено: показать поле "Страна" (обязательно)
  - Если не отмечено: скрыть поле "Страна"

**Шаг 3: Информация о работе**
- Название компании (опционально)
- Должность (опционально)
- **Условные поля**:
  - Чекбокс "Сейчас работаю?"
  - Если отмечено: показать следующие поля (обязательно):
    - Отдел (выпадающий список: Инженерия, Маркетинг, Продажи, HR, Другое)
    - Зарплата (опционально, числовое значение)
  - Если не отмечено: скрыть все поля о работе

**Поведение формы:**
- Используйте компонент stepper/многошаговой навигации
- Валидируйте текущий шаг перед переходом на следующий
- Отображайте сообщения об ошибках валидации для некорректных полей
- Включите кнопку "Назад" для возврата (не на шаге 1)
- Включите кнопку "Далее" для шагов 1-2
- Включите кнопку "Отправить" на последнем шаге
- Шаг 3 должен отображать обзор/сводку всех введенных данных перед отправкой
- **Режим создания**: Отправка создает нового пользователя (API: POST)
- **Режим редактирования**: Заполнение формы данными существующего пользователя, отправка обновляет пользователя (API: PUT)

## Требования к управлению состоянием

Все состояние должно управляться с использованием Effector.

**Необходимые хранилища (stores):**
- Состояние списка пользователей (массив пользователей, состояния загрузки, информация о пагинации, поисковый запрос)
- Состояние формы пользователя (текущий шаг, данные формы, ошибки валидации, режим формы, состояние отправки)

**Необходимые события/эффекты:**
- Загрузить начальных пользователей
- Загрузить больше пользователей (бесконечный скролл)
- Поиск пользователей с дебаунсом
- Валидировать шаг формы
- Перемещаться между шагами формы
- Отправить форму (создание/редактирование)
- Заполнить форму данными существующего пользователя
- Сбросить состояние формы

## Интеграция API

Используйте следующие API endpoints DummyJSON:

**Получить пользователей (пагинировано):**
```
GET https://dummyjson.com/users?limit=10&skip=0
```

**Поиск пользователей:**
```
GET https://dummyjson.com/users/search?q=<query>
```

**Получить одного пользователя:**
```
GET https://dummyjson.com/users/<id>
```

**Создать пользователя:**
```
POST https://dummyjson.com/users/add
```

**Обновить пользователя:**
```
PUT https://dummyjson.com/users/<id>
```

**Примечания по реализации:**
- Используйте Axios для всех HTTP запросов
- Реализуйте надлежащую обработку ошибок при вызовах API
- Показывайте соответствующие состояния загрузки во время запросов
- DummyJSON это mock API; POST/PUT запросы будут возвращать ответы но не будут сохранять данные
